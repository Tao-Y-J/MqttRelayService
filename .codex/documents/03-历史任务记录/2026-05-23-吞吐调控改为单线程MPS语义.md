# 吞吐调控改为单线程MPS语义

## 1. 本次实际修改

- `ThroughputController` 的 `MaxMessagesPerSecond` 语义改为“单线程每秒最大转发量”。
- 限流实现不再把 `MPS` 当成所有工作线程共享的一条全局总上限，而是按“当前活动线程数 × 单线程 MPS”计算全局补充速率。
- 这样当并发线程数提升时，总吞吐上限会线性放大，调控面板中的并发滑块与 MPS 滑块才有组合调控意义。
- `Program.cs` 中 `ThroughputSettingsDto` 注释已同步说明 `MaxMessagesPerSecond` 是单线程语义。

## 2. 当前口径

- 当 `MPS = 0` 时，表示不限制单线程转发速率。
- 当 `MPS > 0` 时，单个活跃工作线程按该值限速；如果当前有多个活跃工作线程，总可用吞吐会随活跃线程数增加。
- 并发滑块仍然控制最多允许多少个工作线程同时处于转发处理中。

## 3. 验证

- 回归执行 `ThroughputControllerTests`，覆盖单消费者限速和多消费者下不再被全局共享桶错误压死的语义。

## 4. 当前边界

- 这次修改的是控制语义和后端行为，没有改 MQTT 主链路、路由策略、重试策略和死信逻辑。
- 前端现有 `maxMessagesPerSecond` 字段名保留不变，只是语义改成“单线程 MPS”，避免扩大 API 兼容面。
