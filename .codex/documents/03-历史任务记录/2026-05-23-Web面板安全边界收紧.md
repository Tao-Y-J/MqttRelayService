# Web 面板安全边界收紧

## 1. 本次实际修改

- 删除 `src/MqttRelayService/wwwroot/index.html` 中的演示模式入口、演示数据生成逻辑和所有 `demoMode` 分支。
- 删除 Dashboard 中的“一键启停”按钮与前端 `toggleForwarding()` 控制逻辑，页面只保留吞吐量与并发参数调节。
- 收紧 `src/MqttRelayService/Program.cs` 里的 `/api/settings/throughput` 接口，请求体和返回体都不再暴露 `isPaused`，接口仅允许更新 `maxMessagesPerSecond` 与 `maxConcurrency`。

## 2. 这样处理的原因

- Web 面板是监控与调参入口，不承担服务启停或远程暂停转发职责。
- 远程暂停转发属于高风险控制动作，保留在页面和 API 中会放大误操作面。
- 演示模式会让页面脱离真实运行状态，不适合作为生产运维面板的长期能力。

## 3. 当前边界

- Dashboard 现在只展示真实指标、真实审计数据和真实客户端状态。
- Dashboard 仍可调整吞吐上限与并发度，但不能通过页面或 API 暂停/恢复转发。
- 服务启停仍然只能通过本地脚本、Windows Service 管理或受控运维流程执行。
