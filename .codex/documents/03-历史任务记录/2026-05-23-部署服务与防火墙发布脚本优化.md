# 部署服务与防火墙发布脚本优化说明

本文档记录了 Windows 服务安装与卸载脚本中防火墙入站规则自动配置的实现方案。

## 1. 目标与实现方案

### 1.1 目标
- 支持在安装 Windows 服务时，将服务占用的三个核心端口（MQTT 1883, API 5000, Dashboard 5001）在 Windows 高级安全 Defender 防火墙中自动配置为允许入站流量访问，实现内网/局域网的高效“发布”。
- 物理清理多余的本地打包源码开发脚本（如 `start-dev.cmd`），转而使用标准的发布包部署。

### 1.2 实际实现
- **install-service.ps1**：
  - 成功创建并启动 Windows 服务后，增加配置 Windows 防火墙入站规则的逻辑。
  - 根据 `appsettings.json` 中实际配置的端口号（`TcpPort` 默认 1883, `MetricsApiPort` 默认 5000, `DashboardPort` 默认 5001），为已启用的服务端口动态配置独立的防火墙入站允许规则（规则命名格式为 `${serviceName}-MQTT`、`${serviceName}-API`、`${serviceName}-Dashboard`）。
  - 创建新规则前执行预清理删除，防止因端口更改或重复安装导致冲突。
  - 顶层捕获防火墙配置异常，在非 Server 系统或未开启防火墙服务的环境下降级提示 Warning，不阻断服务安装主流程。
- **uninstall-service.ps1**：
  - 在服务被停止和删除后，增加清理该服务对应防火墙入站规则的逻辑，保证系统网络环境的干净整洁。
- **物理清理开发脚本**：
  - 从项目根目录中物理删除了 `start-dev.cmd` 与 `stop-dev.cmd` 脚本。

## 2. 验证与交付情况

### 2.1 自动化测试
- 运行 `dotnet test`，97 个单元测试与集成测试全部通过，无退化。

### 2.2 发布包更新
- 运行 `dotnet publish src/MqttRelayService/MqttRelayService.csproj -c Release -r win-x64 --self-contained` 顺利编译并输出到 `bin/Release/net8.0/win-x64/publish/` 目录。
- 经校验，发布目录下的 `Scripts\install-service.ps1` 和 `Scripts\uninstall-service.ps1` 均已成功同步为包含最新防火墙功能的版本。
- 所有 `.ps1` 文件默认应用 `UTF-8 with BOM` 编码进行读写，防止中文显示乱码。
