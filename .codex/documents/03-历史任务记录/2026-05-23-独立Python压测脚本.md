# 独立 Python 压测脚本说明

## 1. 目标与实现方案

- 提供一个不挂接 `src/MqttRelayService` 项目、不修改 `.csproj`、不依赖第三方 Python 包的独立压测脚本，用于直连本机已部署的 MQTT `1883` 端口做并发发布压测。
- 在仓库根目录新增 `stress_mqtt_1883.py`，脚本保持独立，不挂接项目文件；依赖改为独立使用 `paho-mqtt`，安装在仓库外的独立目录 `C:\Users\admin\.cache\codex-mqtt-stress\site-packages`。
- 压测模型采用“8 发 8 收可扩展”的随机单播方式：每个订阅端独占一个目标 Topic，多个发布端并发随机选择目标 Topic 发消息，更接近真实业务场景。
- 为适配每秒万级吞吐目标，脚本使用 MQTT v5 + `loop_start()` 网络循环、二进制固定头载荷、按时长持续压测和低频延迟采样，尽量降低 Python 自身开销。

## 2. 当前交付事实

- 脚本默认连接 `127.0.0.1:1883`，支持通过参数覆盖 `host`、`port`、`topic-prefix`、`publishers`、`subscribers`、`messages`、`payload-size`、`username`、`password`。
- 单条消息负载中写入固定长度二进制头（发送时间戳、序号、发布者索引），订阅端按采样频率计算端到端延迟，输出平均值、P50、P95、P99。
- 脚本默认按 QoS 0 压测 Broker 主链路，依赖不进入仓库项目，只由独立目录提供。
- 默认参数面向高吞吐真实场景：`8` 个发布者、`8` 个订阅者、按 `10` 秒持续发送；如需固定消息数，可显式指定 `--messages`。

## 3. 验证方式

- 使用仓库工作区自带 Python 运行时直接执行脚本，对本机 `127.0.0.1:1883` 做小规模压测样本验证。
- 结果以脚本退出码和控制台摘要为准：若出现连接错误、发送错误或接收量低于期望值，脚本返回非零退出码。
