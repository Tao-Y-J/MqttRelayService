# 默认 SQLite 运行目录自建说明

## 1. 目标与实现方案

- 将审计持久化默认值统一切回 SQLite，不再要求仓库默认携带 SqlServer 连接串。
- 数据库文件改为运行目录下的 `data/audit.db`，不再在项目目录中保留或依赖 `src/MqttRelayService/data/audit.db`。
- 启动时若 SQLite 文件不存在，由代码自动解析到运行目录绝对路径、按需创建目录并初始化表结构。

## 2. 当前交付事实

- `AuditStorageOptions` 默认连接串改为 `Data Source=data/audit.db`。
- `appsettings.json` 与 `appsettings.Development.json` 默认 `Provider` 都为 `Sqlite`。
- `AuditRepository` 创建 `SqlSugarScope` 时会把 SQLite 相对路径转换为 `AppContext.BaseDirectory\data\audit.db` 对应的绝对路径，确保数据库文件始终落在运行目录的 `data` 目录。
- 仓库内原有 `src/MqttRelayService/data/audit.db` 已删除，不再作为默认运行依赖。

## 3. 验证方式

- 运行 `dotnet build src/MqttRelayService/MqttRelayService.csproj` 验证主项目可编译。
- 启动服务后，如运行目录中不存在 `audit.db`，应自动创建并完成表结构初始化。
