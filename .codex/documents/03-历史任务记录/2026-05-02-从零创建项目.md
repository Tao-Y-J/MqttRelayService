## 关键踩坑记录

### 1. MQTTnet 5.x API 变化

- `IMqttServer` → `MqttServer`（无接口）
- `PayloadSegment` → `Payload`（ReadOnlySequence<byte>）
- `ApplicationMessageReceivedHandler` → `InterceptingPublishAsync`
- 所有事件处理器改为 Async 后缀

### 2. 脚本编码问题（重要）

| 现象 | 根因 | 解决 |
|------|------|------|
| cmd 中文乱码 | UTF-8 文件 + cmd 默认 GBK 解码 | cmd 文件改为 **GBK(936)** 编码，`chcp 936` |
| ps1 中文乱码 | UTF-8 无 BOM + PS 5.1 默认 GBK 解码 | ps1 文件改为 **UTF-8 with BOM** |
| 脚本闪退 | `powershell -File` 弹出新窗口执行完立即关闭 | 改用 `powershell -Command "& '...ps1'"` 在当前窗口执行 |

### 3. .csproj 默认包含 Content

SDK 自动包含 `appsettings*.json` 为 Content，显式添加会导致 `NETSDK1022` 重复项错误。解决：删除显式 `<Content Include>` 节点。

### 4. `ReadOnlySequence<byte>.ToArray()` 歧义

MQTTnet 5.x 的 `MqttApplicationMessage.Payload` 是 `ReadOnlySequence<byte>`，直接调用 `.ToArray()` 有歧义。解决：使用 `System.Buffers.BuffersExtensions.ToArray(payload)`。
