# 表格固定列宽与 SqlSugar 审计仓储

## 1. 本次实际修改

- Dashboard 中涉及分页的客户端、连接历史、消息审计、死信审计表格统一改为固定列宽布局，使用 `table-layout: fixed` 与 `colgroup` 固定每列宽度，避免翻页后列宽随内容变化。
- `MqttRelayService.csproj` 引入 `SqlSugarCore`，审计仓储实现统一为 `SqlSugar` 访问层。
- `AuditRepository` 通过 `SqlSugar Queryable`、`Insertable`、`Updateable` 与 `CodeFirst` 完成持久化访问，后续切换正式数据库时只需要替换连接配置与提供程序。

## 2. 当前边界

- 这次数据库层统一的是“访问方式与持久化抽象”，本地开发和正式环境通过配置切换不同数据库提供程序。
- 审计表结构、接口返回形状和现有页面调用方式保持不变。
- 运行中的 `bin/publish` 若未重新替换后端 DLL，则只能先吃到前端固定列宽改动，不能自动获得 SqlSugar 仓储实现。
