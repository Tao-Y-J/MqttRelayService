# 删除重复 Web 监听端口说明

## 1. 目标与实现方案

- 将 Web 管理面的重复监听从“双端口同站点”收敛为“单端口同站点”。
- 统一收敛为 `Web:Enabled` 与 `Web:Port` 两个配置项，由单个 Web 入口端口同时承载 `/api/*` 指标接口与 Dashboard 前端页面。
- 删除旧的 `MetricsApiPort`、`EnableMetricsApi`、`EnableDashboard`、`DashboardPort` 配置与安装脚本中的重复防火墙规则，避免 5000 和 5001 同时暴露同一套页面与接口。

## 2. 当前交付事实

- `Program.cs` 现在只为 Web 面板注册一个 Kestrel 监听端口。
- `WebOptions` 改为仅保留 `Enabled` 与 `Port`，`appsettings.json` 与 `appsettings.Development.json` 同步切换到新配置结构。
- `install-service.ps1` 改为只发布一个 Web 端口规则，并在安装结果中统一输出 `Web 入口` 与 `指标 API`。
- `README.md` 与仓库代码地图已同步更新为单 Web 端口事实。

## 3. 验证方式

- 运行 `dotnet build src/MqttRelayService/MqttRelayService.csproj` 验证主项目可编译。
- 运行后应只在配置的单个 Web 端口上访问 Dashboard 与 `/api/metrics`。
