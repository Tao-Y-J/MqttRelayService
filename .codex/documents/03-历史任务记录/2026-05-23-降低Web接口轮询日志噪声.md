# 降低 Web 接口轮询日志噪声

## 1. 实际修改

- `SerilogLogging.CreateLogger` 之前只读取 `Serilog:MinimumLevel:Default`，没有把 `Serilog:MinimumLevel:Override` 注册到实际 Logger。
- 本次补上 `Override` 遍历逻辑，使 `appsettings.json` 中已有的分类级别配置真正生效：
  - `Microsoft = Warning`
  - `Microsoft.Hosting.Lifetime = Information`
- 修复后，`/api/metrics`、`/api/messages` 等由 `Microsoft.AspNetCore.*` 输出的请求开始/结束、执行端点、写 JSON 响应等信息级日志将被抑制，不再刷屏。

## 2. 结果边界

- 本次没有关闭业务代码自己的 `Information` 日志。
- 本次没有移除 ASP.NET Core 请求日志功能，只是让现有分类覆盖配置按预期工作。
- `Warning`、`Error`、`Fatal` 级别的框架日志仍然保留。
