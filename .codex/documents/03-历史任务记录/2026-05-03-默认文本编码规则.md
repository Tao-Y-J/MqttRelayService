# 2026-05-03 默认文本编码规则

## 目标

把中文环境下的文本文件默认编码写成仓库长期规则，减少 Windows / PowerShell 场景中的乱码。

## 实际修改

- `AGENTS.md` 新增仓库级文本编码要求：除明确例外外，文本文件读写默认使用 `UTF-8 with BOM`。
- `lessons.md` 把 Windows 编码经验扩展为仓库文本编码基线，明确 `.cmd` / `.bat` 是 `GBK (936)` 例外。
- `.codex/documents/01-长期有效规范/仓库上下文管理入口.md` 同步长期有效编码约束。
- 上述长期文档实际写回为 `UTF-8 with BOM`。

## 关键决策

- 默认文本文件统一使用 `UTF-8 with BOM`，因为仓库长期维护中文文本，并且经常通过 Windows / PowerShell 读取和修改文件。
- `.cmd` / `.bat` 保持 `GBK (936)`，因为 cmd 对 UTF-8 中文脚本的兼容性不足，直接改为 UTF-8 会破坏执行。

## 验证

- 检查 `AGENTS.md`、`lessons.md` 和 `.codex/documents/01-长期有效规范/仓库上下文管理入口.md` 的文件头，确认带有 UTF-8 BOM。
- 重新读取上述文档，确认新增中文内容可读、无乱码。

## 边界

- 本次只更新协作规则和文档编码，不修改业务代码、MQTT 主链路和安装脚本逻辑。
- 本次未批量转换仓库内全部文本文件，只处理本次修改涉及的长期文档。
