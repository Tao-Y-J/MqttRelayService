# 审计日志新增是否命中订阅者状态列

## 1. 本次实际修改

- **Models / MessageContext**：新增 `CurrentIsSubscriberHit` 的 `AsyncLocal<bool?>` 上下文属性，支持命中状态沿着异步调用链透传。
- **Models / MessageAuditRecord & MessageLogEntry**：分别新增 `IsSubscriberHit` 布尔类型属性，用于在 SQLite 数据库表 `message_audit` 中物理持久化并在内存中跟踪该状态。
- **Services / IMetricsService & MetricsService**：
  - 修改 `RecordForwarded` 接口及实现方法签名，添加 `bool isSubscriberHit = false` 可选参数。
  - 在 `AddOrUpdateLog` 更新分支中，新增同步复制 `existing.IsSubscriberHit = entry.IsSubscriberHit`。
  - 构造 `MessageLogEntry` 和 `MessageAuditRecord` 时，将透传的命中状态存入对应属性中。
- **Services / MessageDeliveryService**：
  - 修改私有方法 `TryForwardAsync` 签名，添加 `bool isSubscriberHit` 参数，并在注入前将其写入 `MessageContext.CurrentIsSubscriberHit`；在 `finally` 块中确保进行彻底置空。
  - 在 `ProcessMessageAsync` 中，直接在过滤匹配完毕后将 `targets.Count > 0` 作为命中参数传入 `TryForwardAsync`。并且移除了原先的“零目标订阅者直接返回”旁路优化，统一进行 Broker 注入，以确保所有经过处理的消息都会产生完整一致的审计状态、端到端延迟和命中记录。
- **Services / MetricsMqttBrokerHost (装饰器)**：
  - 在 `PublishAsync` 拦截器中，同步读取 `MessageContext.CurrentIsSubscriberHit.Value`。
  - 触发 `RecordForwarded` 指标回调时将该命中状态完整传出。
- **Web 前端管理面板 (index.html)**：
  - 调整“最新转发消息队列”表格与“转发消息物理审计日志”表格的 `<colgroup>` 宽度和表头，在状态列后面新增一列“命中”列。
  - 行渲染逻辑中，利用全局新增的辅助格式化函数 `getHitBadge(log)` 动态渲染：未决状态显示 `-`，最终状态（`Succeeded`/`Failed`/`DeadLetter`）根据布尔属性显示绿色带勾的 `命中` 或红色带叉的 `未命中` Badge。
  - 在右侧消息抽屉详情板中，在执行状态下部新增 `是否命中: [命中 / 未命中]` 属性展示。
- **Tests / DashboardMetricsTests & MessageDeliveryServiceTests**：
  - **Moq 表达式树兼容 (CS0854)**：由于 `RecordForwarded` 引入了可选参数，Moq 的 `Verify` 内部表达式树在编译时会因为可选参数报错 `CS0854`。本次修复在 `DashboardMetricsTests.cs` 中对所有的 `Verify(m => m.RecordForwarded(...))` 表达式均显式传入了第五个 `It.IsAny<bool>()` 占位符，从而绕过 C# 编译限制。
  - **反射形参补全**：修正了 `MessageDeliveryServiceTests.cs` 中反射调用 `TryForwardAsync` 时的实参数组，补全了 `isSubscriberHit = true` 布尔项，实现 100% 单元测试自愈。

## 2. 技术决策与边界

- **统一 Broker 注入**：移除零目标订阅者的旁路返回，实现消息全部由 `PublishAsync` 转发。此决策确保了不论消息是否有客户端订阅，都能生成一条完全对应的审计日志记录（状态统一转为 `Succeeded`，并且命中状态显示为 `未命中`），解决了旁路优化导致“无订阅者消息状态悬挂在 Queued”的体验漏洞。
- **SqlSugar CodeFirst 自愈**：SqlSugar 具有表结构自愈能力，程序启动时会自动检测并在 `message_audit` 物理表中追加 `IsSubscriberHit` 列，无需手动编写 SQL 数据库迁移脚本。
- **Moq 测试规范**：由于 Moq 的 `Verify` / `Setup` 仅支持显式表达式，任何接口可选参数变化都必须在测试断言中显式填补，今后应严格防范 CS0854 编译陷阱。
