# 2026-05-15 Retained消息转发修复

## 实际修改

- `RouteContext` 增加 `Retain` 标志，用于保存客户端发布消息的 MQTT Retained Message 语义。
- `IMqttBrokerHost.PublishAsync` 增加 `retain` 参数，`MqttBrokerHost.PublishAsync` 注入消息时调用 `WithRetainFlag(retain)`。
- `MqttBrokerHost.OnInterceptingPublishAsync` 在拦截客户端原始发布时保存 `ApplicationMessage.Retain`。
- `MessageDeliveryService` 转发时传递 `context.Retain`；当 `Retain=true` 且当前没有匹配订阅者时，仍然注入 Broker，让 MQTTnet 保存运行期内存 retained 消息。
- `MqttBrokerIntegrationTests` 增加迟到订阅者接收 retained 消息的端到端测试。

## 决策与边界

- 当前实现支持 MQTTnet Broker 进程运行期内存 retained 语义。
- 当前实现不提供 retained 消息的磁盘持久化；服务进程重启后 retained 消息不会恢复。
- 非 retained 消息在当前没有匹配目标客户端时仍按原逻辑直接视为处理成功。
