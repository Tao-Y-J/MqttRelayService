# 2026-05-03 安装卸载脚本异常修复

## 目标

修复 Windows Service 安装和卸载脚本在实际部署路径中的异常。

## 实际修改

- `install-service.ps1` 注册服务时使用带双引号的 `BinaryPathName`，确保服务可执行文件路径按完整路径写入 Windows Service。
- `uninstall-service.ps1` 删除服务时使用 `sc.exe delete`，避免依赖 PowerShell 5.1 中不存在的 `Remove-Service` 命令。
- `uninstall-service.ps1` 停止服务时按非 `Stopped` 状态统一进入停止流程，删除服务后等待服务对象消失。

## 边界

- 本次修改只处理安装和卸载脚本，不修改 Worker Service 运行逻辑。
- 本次修改不改变服务名称、显示名称、启动类型和配置读取规则。
- 本次会话未执行真实安装和卸载操作，因为该操作会改写当前机器的 Windows Service 注册表状态。

## 验证

- `install-service.ps1` 和 `uninstall-service.ps1` 通过 PowerShell AST 语法解析。
- 确认 PowerShell 5.1 环境下未找到 `Remove-Service` 命令。
- 确认 `.ps1` 脚本保留 UTF-8 BOM。
- 确认 `.cmd` 脚本按 GBK 解码后中文可读。
- 执行 `dotnet build MqttRelayService.slnx`，构建成功。
- 执行 `dotnet test MqttRelayService.slnx --no-build`，62 个测试全部通过。
