## 3. 关键决策与约束（长期有效）

### 3.1 文本文件编码规范（不可变更）

仓库中文文本长期运行在 Windows / PowerShell 环境下，编码规则已踩坑确认：

| 范围 | 编码 | 原因 |
|----------|------|------|
| 默认文本文件（`.md`、`.ps1`、`.json`、`.yml`、`.xml`、`.cs` 等） | **UTF-8 with BOM** | 中文内容在 Windows / PowerShell 5.1 下读取更稳定，无 BOM 时容易按系统编码（GBK）解码并出现乱码 |
| `.cmd` / `.bat` | **GBK(936)** | cmd 默认用 GBK 解码，UTF-8 会导致中文乱码或命令解析异常 |

> 新增或修改文本文件时，除仓库内已有明确例外外，默认写回 `UTF-8 with BOM`。修改 `.cmd` / `.bat` 时必须保持 `GBK(936)`。

### 3.2 MQTTnet 5.x API 适配要点

- `MqttServerFactory.CreateMqttServer()` 返回 `MqttServer`（不是 `IMqttServer`）
- 事件全面异步化：`ValidatingConnectionAsync`、`InterceptingPublishAsync`、`ClientConnectedAsync` 等
- `Payload` 属性是 `ReadOnlySequence<byte>`，需用 `System.Buffers.BuffersExtensions.ToArray()` 转换
- `ClientDisconnectedEventArgs` 没有 `Reason` 属性，使用 `DisconnectType`

### 3.3 可靠性设计原则

- 所有后台入口必须顶层 `try/catch`
- MQTT 事件回调只做轻量操作（入队即返回）
- 默认"至少一次"转发语义，允许重复
- 消息唯一标识 `MessageId`（GUID）
- 有界队列（`System.Threading.Channels`），禁止无界增长
- 超过最大重试次数进入死信文件（JSON 格式）

### 3.4 C# 命名空间风格

- 全仓库 `.cs` 文件统一使用块级命名空间写法：`namespace MqttRelayService.Utilities { ... }`
- 不使用文件作用域命名空间写法：`namespace MqttRelayService.Utilities;`
- 新增或修改 C# 文件时，必须保持块级命名空间风格
