## 2. 核心数据流

```
 客户端发布消息
  → MqttBrokerHost.InterceptingPublishAsync（轻量：入队 + ProcessPublish=false）
  → InMemoryMessageQueue.EnqueueAsync（有界 Channel，满载显式丢弃）
  → MessageDeliveryService.ConsumeLoopAsync（后台循环）
    → ReadAllAsync（ChannelReader 异步挂起等待）
    → MessageRouter.RouteAsync（Topic 匹配，异常向上传播）
    → IMqttBrokerHost.PublishAsync（InjectApplicationMessage，单次注入）
    → 成功：状态=Succeeded / 失败：退避等待 → 重试 → 死信
```

## 3. 依赖关系

```
Program.cs
  ├─ BrokerWorker → IMqttBrokerHost
  ├─ QueueMetricsWorker → IMessageQueue
  └─ DeliveryWorker → IMessageDeliveryService（Host 生命周期自动管理）

MqttBrokerHost
  ├─ IAuthService（ValidatingConnectionAsync）
  ├─ IClientRegistry（Connected/Disconnected/Subscription）
  └─ IMessageQueue（InterceptingPublishAsync 入队）

MessageDeliveryService
  ├─ IMessageQueue（消费）
  ├─ IMessageRouter（路由）
  ├─ IMqttBrokerHost（转发注入）
  ├─ IRetryPolicyProvider（重试间隔）
  └─ IDeadLetterService（死信写入）
```

## 4. 配置绑定关系

| Options 类 | appsettings.json 节 | 使用位置 |
|-----------|-------------------|---------|
| ServiceOptions | Service | Program.cs（ServiceName）、SerilogLogging（扩展属性） |
| MqttOptions | Mqtt | MqttBrokerHost（端口） |
| AuthOptions | Auth | AuthService（认证逻辑） |
| RoutingOptions | Routing | MessageRouter（EchoToSender） |
| ReliabilityOptions | Reliability | InMemoryMessageQueue、MessageDeliveryService、DeadLetterService、RetryPolicyProvider |

## 5. 关键文件修改频率预期

| 文件 | 预期修改场景 |

| `appsettings.json` | 每次部署环境变更 |
